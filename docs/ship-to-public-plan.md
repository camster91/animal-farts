# PootBox — Ship to Public — v1.0 (post-test) Plan

## Goal

Make PootBox reachable by anyone (not just Cam's iPhone), so a real audience can find it, install it, use it. Two parallel tracks: web PWA (faster, broader) + Android Play Store (slower, more discoverable).

**Current state:** live at https://animals.ashbi.ca. Working, tested, simplified. APK v2.0 on GitHub. Recording on iPhone works. Recording on Android requires the Capacitor APK.

## Track 2: Android Play Store (reach: Play Store search, 2-day install friction, $25 fee)

**Timeline: 1 week after kid test passes + Play Store dev account + privacy policy on a real domain.**

### What's already done
- ✅ Capacitor 7 project at `/android`
- ✅ `RECORD_AUDIO` permission in AndroidManifest
- ✅ APK builds (10s, 71MB debug, ~12MB release with R8)
- ✅ `app-debug.apk` released on GitHub (v2.0)
- ✅ App ID: `com.ashbi.pootparty`

### What's MISSING for Play Store

1. **Release keystore** — generate `poot-party.keystore`, add to gitignore, configure `signingConfigs.release` in `android/app/build.gradle`
2. **AAB (Android App Bundle)** — Play Store requires `.aab` not `.apk`. `./gradlew bundleRelease` produces this
3. **Play Store developer account** — $25 one-time fee, takes 48h to verify
4. **App icon** — 512x512 PNG with no transparency. The current icons are copies of the 💨 emoji. Needs proper design.
5. **Feature graphic** — 1024x500 banner for the Play Store listing
6. **Screenshots** — at least 2 per device type (phone, 7" tablet, 10" tablet)
7. **Store listing text**:
   - Short description (80 chars)
   - Full description (4000 chars)
   - "What's new" text
   - Categorization: Family / Casual / Music
   - Content rating: Everyone / PEGI 3
8. **Privacy policy URL** — must be on a public, crawlable URL. Currently at https://animals.ashbi.ca/privacy.html but the page is generated by the SPA. Play Store wants a static URL. Either:
   - Move to `https://ashbi.ca/pootbox/privacy.html` (a separate static page)
   - Or accept the current URL
9. **App signing by Google** — optional but recommended. Play App Signing means the AAB is signed with a separate key after upload.
10. **Data safety form** — Play Store asks: does the app collect data, share with third parties, etc. We collect microphone (with permission), store locally. Form answers: yes, audio recordings, only on device, no third parties, encrypted in transit (HTTPS), can be deleted.
11. **Target API level** — currently 34. Play Store 2024 requires 34+. ✓
12. **App-ads.txt** — only if you run ads. We don't.
13. **Content rating questionnaire** — IARC questionnaire on Play Console. Will probably rate "Everyone."
14. **Target audience** — "Children under 13" flag — this affects COPPA compliance. We're collecting audio which is "personal info" so YES we need to flag this. Adds friction to Play Store review.
15. **Pre-launch report** — automated checks Play runs before review

### Tasks for Play Store ship (in priority order)

**Day 1 (3-4 hours):**
1. Create a release keystore (`keytool -genkey -v -keystore poot-party.keystore ...`)
2. Configure `signingConfigs.release` in `android/app/build.gradle`
3. Configure `buildTypes.release` to sign with the release config
4. Build and verify the signed release APK: `./gradlew assembleRelease`
5. Build the AAB: `./gradlew bundleRelease` (output: `app/build/outputs/bundle/release/app-release.aab`)
6. Test the signed release APK on a real Android device (sideload, verify mic works)

**Day 2 (4-6 hours):**
7. Design a real app icon (512×512 PNG, no transparency) — 1-2 hours with designer
8. Create a feature graphic (1024×500) — 1 hour
9. Take screenshots — 4-6 of the kid test session. Mock them up with text overlays.
10. Write the Play Store listing text (short + long description)

**Day 3 (2-4 hours):**
11. Set up Google Play Developer account ($25)
12. Fill out the Data Safety form
13. Fill out the Content Rating questionnaire
14. Target audience: decide on "Children" flag (see below)
15. Upload the AAB as an internal testing track

**Day 4-7:**
16. Wait for Play Store review (24-48h for internal testing)
17. Promote to production
18. Public listing is live

### COPPA decision (important)
Because we record audio (which is "personal info" for COPPA), Play Store requires us to either:
- **A)** Mark the app as "designed for children" — gets a "Made for kids" badge, but limits monetization and features
- **B)** Mark as "not for children" but require a gate — we already have a 4-digit parent PIN gate, which Play considers sufficient for "mixed audience" apps
- **C)** Mark as "general audience" with a privacy policy — risky, could get rejected

**Recommended: B** — we already have the PIN-gated /parent dashboard. Document this in the privacy policy.

## Risk register

- **COPPA compliance risk** — recording audio from children is sensitive. The PIN-gated /parent dashboard should satisfy Play Store, but rejection is possible. Plan B: remove the recording feature entirely and ship without it.
- **iOS App Store** — not in this plan. PWA on iPhone Safari is the workaround. Adding iOS via Capacitor is similar to Android but needs Apple Developer account ($99/yr) and Mac for Xcode builds.
- **Server cost** — currently on Coolify self-hosted, no cost. If we get 100+ users, the SQLite + uploads directory will need to scale. For now: fine.
- **Hostile use** — kids sharing recordings of their voices publicly. We have an "Adult Mode" flag in /parent but not on the kid's side. Mitigation: keep recordings local-only, no public sharing. We're already doing this.
- **DMCA / inappropriate content** — kids could record sounds and share. We don't have a public feed anymore (the v1 explore page is gone), so this is mostly contained.
